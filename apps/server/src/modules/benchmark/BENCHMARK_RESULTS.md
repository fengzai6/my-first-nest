# Redis 缓存性能测试报告

测试时间: 2026-05-15  
测试环境: 本地开发 (localhost)

## 测试配置

- **Redis**: Docker 容器 (redis:8-alpine), 端口 6379
- **数据库**: PostgreSQL (本地)
- **测试数据**: 2 个用户

## 测试结果

### 1. 单次查询对比

| 指标 | 数据库 | Redis 缓存 | 差异 |
|------|--------|-----------|------|
| 耗时 | 6.79ms | 4.34ms | - |
| 性能提升 | - | - | **1.6x** |

### 2. 批量查询对比 (100 次)

| 指标 | 数据库 | Redis 缓存 | 差异 |
|------|--------|-----------|------|
| 总耗时 | 197.36ms | 18.68ms | - |
| 平均耗时 | 1.97ms | 0.19ms | - |
| 性能提升 | - | - | **10.6x** |

### 3. 场景测试 (冷启动 vs 热缓存)

| 指标 | 无缓存 (冷启动) | 有缓存 (热缓存) | 差异 |
|------|----------------|----------------|------|
| 总耗时 | 3.73ms | 0.74ms | - |
| 平均耗时 | 1.87ms | 0.37ms | - |
| 性能提升 | - | - | **5.0x** |
| 响应时间减少 | - | - | **80.1%** |

## 结论

1. **批量查询场景**: Redis 缓存优势最为明显，性能提升达 **10.6x**
2. **单次查询场景**: 提升较小 (1.6x)，因为本地数据库响应快
3. **生产环境**: 网络延迟更高时，Redis 的优势会更显著
